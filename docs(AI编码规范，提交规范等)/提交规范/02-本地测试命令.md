# 本地测试命令（与 CI 对齐）

在仓库根目录 `D:\Code-Dev\BaiShou-Next` 执行。Node 版本建议与 CI 一致：**22.x**；包管理器：**pnpm 10**。

## 一键跑完全部 CI 检查

PowerShell：

```powershell
cd D:\Code-Dev\BaiShou-Next

pnpm install
pnpm typecheck
pnpm turbo run test --continue
pnpm --filter @baishou/desktop exec eslint -c ../../eslint.desktop.ci.mjs . --cache --quiet
pnpm --filter @baishou/mobile exec eslint -c ../../eslint.mobile.ci.mjs . --cache --quiet
pnpm format:check
```

任一步失败即停止排查，无需先 push。

## 分项说明

| CI Job         | 本地命令                                                                                    |
| -------------- | ------------------------------------------------------------------------------------------- |
| Typecheck      | `pnpm typecheck`                                                                            |
| Test           | `pnpm turbo run test --continue`                                                            |
| Lint (desktop) | `pnpm --filter @baishou/desktop exec eslint -c ../../eslint.desktop.ci.mjs . --cache --quiet`                              |
| Lint (mobile)  | `pnpm --filter @baishou/mobile exec eslint -c ../../eslint.mobile.ci.mjs . --cache --quiet` |
| Prettier       | `pnpm format:check` / 修复：`pnpm format`                                                   |

## 只验证本次改动相关包

```powershell
pnpm --filter @baishou/core test
pnpm --filter @baishou/desktop typecheck
pnpm --filter @baishou/ai test
```

## better-sqlite3 与集成测试

若出现 `NODE_MODULE_VERSION` 不匹配：

```powershell
pnpm rebuild better-sqlite3
```

仍失败时，依赖真实 SQLite 的 desktop 集成测会自动 **skip**（见 `apps/desktop/src/main/services/__tests__/better-sqlite3-available.ts`）。**GitHub Actions（Ubuntu + Node 22）上会正常执行这些测试。**

## Mobile Lint 说明

移动端使用根目录 [`eslint.mobile.ci.mjs`](../../eslint.mobile.ci.mjs)：对存量未使用变量为 **warn**，`eslint --quiet` 时不阻断；并忽略 `scripts/`、`mocks/`、`.expo/` 等。日常开发仍可用：

```powershell
pnpm --filter @baishou/mobile run lint
```

## 推送

本地全部通过后再：

```powershell
git push origin main
```

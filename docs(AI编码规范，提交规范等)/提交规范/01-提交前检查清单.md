# 提交前检查清单

在 `git push origin main` 或创建 PR 之前，请按顺序完成下列项。全部通过后再推送，可避免在 GitHub Actions 上长时间等待失败结果。

## 必做（与 CI 一致）

- [ ] **安装依赖**：`pnpm install`（锁文件变更后必须重装；若集成测报 better-sqlite3 二进制错误，可执行 `pnpm rebuild better-sqlite3`）
- [ ] **Typecheck**：`pnpm typecheck`
- [ ] **Test**：`pnpm turbo run test --continue`
- [ ] **Lint Desktop**：`pnpm --filter @baishou/desktop exec eslint . --cache --quiet`
- [ ] **Lint Mobile**：`pnpm --filter @baishou/mobile exec eslint -c ../../eslint.mobile.ci.mjs . --cache --quiet`
- [ ] **Prettier**：`pnpm format:check`（未通过可运行 `pnpm format` 后只提交格式化相关变更）

## 建议

- [ ] 单次提交聚焦一个主题；大重构拆成多个可读的 commit（参见 [03-Commit-Message-规范.md](./03-Commit-Message-规范.md)）
- [ ] 不提交一次性脚本（如 `count-code-lines.mjs`、`split-*.ps1`），用完即删
- [ ] 推送前执行 `git status`，确认无 `.env`、密钥、本地数据库等敏感文件

## GitHub 分支保护（可选）

若刚启用 CI，建议先在仓库 Settings → Branches 中仅勾选 **Typecheck** 为 required；待 Lint / Prettier / Test 全绿后再逐步加上其余 job。

## 若本地仍有 5 个 Desktop Lint 错误

部分文件若被 IDE 占用无法自动修复，请关闭相关标签页后执行：

```powershell
node scripts/fix-ci-gate.mjs
```

或手动按 [02-本地测试命令.md](./02-本地测试命令.md) 中的说明修改未使用变量。
